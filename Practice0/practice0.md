_**DDD:**_  

**Контексты (Bounded Contexts)**
1. Управление пользователями (User Management):
    - Регистрация и аутентификация пользователей (студенты, преподаватели).
    - Управление ролями и правами доступа.
2. Управление лабораторными работами (Lab Management):
    - Загрузка и хранение лабораторных работ.
    - Связь работ с конкретными студентами и преподавателями.
3. Оценка кода (Code Evaluation):
    - Автоматическая проверка кода на соответствие критериям.
    - Генерация отчётов с оценками и рекомендациями.
4. Статистический анализ (Code Metrics):
    - Расчёт метрик кода (тестовое покрытие, сложность, читаемость и т.д.).
    - Интеграция метрик в отчёты.
5. Чат-бот (Telegram Bot):
    - Интерфейс для взаимодействия с системой (запрос отчётов, загрузка кода).
6. Интеграция с GitHub (GitHub Integration):
    - Загрузка кода из репозиториев GitHub.
    - Синхронизация данных с системой.

**Сущности (Entities)**
1. Пользователь (User):
   - Атрибуты: ID, имя, роль (студент/преподаватель), email, пароль.
   - Методы: аутентификация, регистрация.
2. Лабораторная работа (Lab):
   - Атрибуты: ID, название, описание, срок сдачи, статус (сдана/не сдана).
   - Методы: загрузка кода, обновление статуса.
3. Код (Code):
   - Атрибуты: ID, содержимое (текст кода), ссылка на репозиторий GitHub.
   - Методы: парсинг, анализ.
4. Отчёт (Report):
   - Атрибуты: ID, оценка, рекомендации, метрики кода.
   - Методы: генерация отчёта, отправка пользователю.
5. Метрики кода (Code Metrics):
   - Атрибуты: тестовое покрытие, сложность, читаемость.
   - Методы: расчёт метрик.

**Агрегаты (Aggregates)**
1. Пользователь (User Aggregate):
   - Корневая сущность: User.
   - Включает: роли, права доступа.
2. Лабораторная работа (Lab Aggregate):
   - Корневая сущность: Lab.
   - Включает: Code, Report.
3. Отчёт (Report Aggregate):
   - Корневая сущность: Report.
   - Включает: Code Metrics, рекомендации.
  
**Сервисы (Services)**
1. Code Evaluation Service:
   - Проверка кода на соответствие критериям.
   - Генерация отчётов.
2. Code Metrics Service:
   - Расчёт метрик кода (тестовое покрытие, сложность, читаемость).
3. GitHub Integration Service:
   - Загрузка кода из репозиториев GitHub.
   - Синхронизация данных с системой.
4. Telegram Bot Service:
   - Обработка запросов от пользователей.
   - Отправка отчётов и рекомендаций.
      
Глоссарий (Ubiquitous Language)
- Код: Исходный текст программы на языке C#.
- Отчёт: Документ, содержащий оценку,комментарии по выделенным критериям.
- Метрики кода: Показатели, такие как тестовое покрытие, сложность, читаемость.
- Лабораторная работа: Задание, которое студент должен выполнить и сдать.
